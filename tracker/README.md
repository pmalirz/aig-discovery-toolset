# Tracker — Per-Engagement Working Directory

This directory contains the live data for a specific client engagement. All filled templates, scorecards, and generated reports are stored here.

## Directory Structure

```
tracker/
├── engagement-tracker.md          # Central progress & coordination document
├── company-profile.md             # Filled company profile from executive workshop
├── entities/                      # Legal entity profiles (if multi-entity organization)
│   └── <entity-name>.md
├── pillars/                       # Organized by business pillar / division
│   └── <pillar-name>/
│       ├── teams/                 # Filled team cards
│       │   └── <team-name>-team-card.md
│       ├── capabilities/          # Mapped business capabilities
│       │   └── <capability-name>.md
│       └── ideas/                 # Submitted AI use ideas
│           └── <idea-name>.md
├── scorecards/                    # Scored AI opportunity assessments (cross-cutting)
│   └── <idea-name>-scorecard.md
└── reports/                       # Generated analysis and executive deliverables
    ├── ai-readiness-heatmap.md
    ├── executive-summary.md
    └── togaf-matrices.md
```

## Getting Started

1. Copy `templates/engagement-tracker.md` to `tracker/engagement-tracker.md`
2. Copy `templates/company-profile.md` to `tracker/company-profile.md`
3. Create pillar directories under `tracker/pillars/` matching the organization's structure
4. Fill templates during workshops and save them to the appropriate subdirectory
5. Run agent skills (`aig-validate`, `aig-assess`, `aig-heatmap`, `aig-matrix`) to generate scorecards and reports

## File Naming Convention

- Use **kebab-case** for all file and directory names
- Team cards: `<team-name>-team-card.md` (e.g., `claims-processing-team-card.md`)
- Capabilities: `<capability-name>.md` (e.g., `claims-intake-registration.md`)
- Ideas: `<idea-name>.md` (e.g., `intelligent-claims-intake.md`)
- Scorecards: `<idea-name>-scorecard.md` (e.g., `intelligent-claims-intake-scorecard.md`)

## Important Notes

- This directory contains **client-sensitive data**. Handle access controls appropriately.
- All files should be committed to Git for version tracking and auditability.
- The `reports/` directory is regenerated by agent skills — manual edits to generated reports will be overwritten.
