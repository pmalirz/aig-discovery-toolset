# Tracker — Per-Engagement Working Directory

This directory contains the live data for a specific client engagement. It holds both the raw **source documents** shared by teams during interviews and the **md silver layer** — the structured Markdown + YAML files that all assessment, scoring, and reporting skills operate on.

## Directory Structure

```
tracker/
├── engagement-tracker.md          # Central progress & coordination document
├── company-profile.md             # Filled company profile from executive workshop
├── entities/                      # Legal entity profiles (if multi-entity organization)
│   └── <entity-name>.md
├── pillars/                       # Organized by business pillar / division
│   └── <pillar-name>/
│       ├── sources/               # Raw unstructured docs shared by teams (docx, xlsx, pptx, pdf)
│       ├── teams/                 # Filled team cards (silver layer)
│       │   └── <team-name>-team-card.md
│       ├── capabilities/          # Mapped business capabilities (silver layer)
│       │   └── <capability-name>.md
│       └── ideas/                 # Submitted AI use ideas (silver layer)
│           └── <idea-name>.md
├── scorecards/                    # Scored AI opportunity assessments (cross-cutting)
│   └── <idea-name>-scorecard.md
└── reports/                       # Generated analysis and executive deliverables
    ├── ai-readiness-heatmap.md
    ├── executive-summary.md
    └── togaf-matrices.md
```

### Data Flow: Sources → Silver Layer → Reports

1. **Sources (raw input):** Teams share unstructured documents during interviews and discovery. The EA/consultant stores them in `pillars/<pillar>/sources/` by convention, organized by what they describe.
2. **Silver layer (structured md):** The `aig-interview` skill translates source documents into structured Markdown + YAML (team cards, capabilities, ideas). These are the files that live in `teams/`, `capabilities/`, and `ideas/`.
3. **Validation & scoring:** `aig-validate` and `aig-assess` operate on the silver layer to check completeness and generate scorecards.
4. **Reports:** `aig-matrix` and `aig-heatmap` aggregate silver-layer data into executive deliverables.

## Getting Started

1. Copy `templates/engagement-tracker.md` to `tracker/engagement-tracker.md`
2. Copy `templates/company-profile.md` to `tracker/company-profile.md`
3. Create pillar directories under `tracker/pillars/` matching the organization's structure
4. Create a `sources/` directory under each pillar for collecting unstructured documents from teams
5. Fill templates during workshops and save them to the appropriate subdirectory
6. Run agent skills (`aig-validate`, `aig-assess`, `aig-heatmap`, `aig-matrix`) to generate scorecards and reports

## File Naming Convention

- Use **kebab-case** for all file and directory names
- Team cards: `<team-name>-team-card.md` (e.g., `claims-processing-team-card.md`)
- Capabilities: `<capability-name>.md` (e.g., `claims-intake-registration.md`)
- Ideas: `<idea-name>.md` (e.g., `intelligent-claims-intake.md`)
- Scorecards: `<idea-name>-scorecard.md` (e.g., `intelligent-claims-intake-scorecard.md`)

## Important Notes

- This directory contains **client-sensitive data**. Handle access controls appropriately.
- All files should be committed to Git for version tracking and auditability.
- The `sources/` directories contain raw documents shared by teams — keep them as-is for provenance and traceability.
- The `reports/` directory is regenerated by agent skills — manual edits to generated reports will be overwritten.
